## Notes for Database Import

When adding these to the `work_items` table:

1. **Video files** should use `hero_media_url` pointing to the file path
2. **Thumbnails** should use `thumbnail_url` pointing to the thumbnail file
3. **Category** should be set to `'creative'` for most of these
4. **Project type** examples: "Video Campaign", "Product Video", "Event Video", "AI Advertisement"
5. **Status** should be `'published'` to show on the site

## Recommended Work Items Structure

### Example 1: 11PC Campaign
```json
{
  "client_name": "11PC",
  "project_title": "11PC Launch Campaign",
  "project_type": "Video Campaign",
  "category": "creative",
  "thumbnail_url": "/portfolio/11PC Launch.jpg",
  "hero_media_url": "/portfolio/11PC Love All.mp4",
  "hero_media_type": "video",
  "status": "published",
  "featured": true
}
```

## Quick Reference for Database Import

### Vimeo Videos (Use Vimeo Embed)
- Use `hero_media_type: 'video'` with Vimeo embed URL
- Format: `https://player.vimeo.com/video/{VIDEO_ID}`
- Or use Vimeo ID directly in frontend components

### Local Videos (Use File Path)
- Use `hero_media_url: '/portfolio/{filename}.mp4'`
- Set `hero_media_type: 'video'`

### Thumbnails
- Vimeo videos: Use mapped thumbnail paths from above
- Local videos: Use related thumbnail or generate from video
